# AGENT_CODE_STYLE

唯一标准：

一个完全不了解项目的人，只看 notebook，就应该能：

1. 看懂这个实验要回答什么问题
2. 看懂实验是怎么做的
3. 找到关键参数和变量
4. 直接在 notebook 里改实验
5. 不需要先读很多 `.py` 文件

因此必须遵守：

- notebook 是实验本体，不是调用器
- 实验主流程必须显式写在 notebook 中
- notebook 里的代码要像伪代码一样自我解释
- 关键比较、变量扫描、指标计算入口必须在 notebook 中可见
- `.py` 只放底层工具，不要把实验逻辑藏进去

禁止：

- `run_everything(cfg)` 这种黑盒式 notebook
- 读 notebook 时看不出实验到底做了什么
- 想改实验时必须跳进多个 `.py` 文件
- 为了“工程整洁”牺牲 notebook 可读性

允许放进 `.py` 的内容：

- 通用数学运算
- 投影、分解、绘图等底层函数
- 可复用的数据结构

不允许放进 `.py` 的内容：

- 实验在比较什么
- 实验按什么维度扫描
- 实验每一步如何串起来

一票否决：

如果 notebook 不能让陌生读者直接读懂并修改实验，那么实现就不合格。
